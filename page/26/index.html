谈一谈面向对象思想

  1. 什么是面向对象？面向对象是相对于面向过程来说的，面向过程强调的是功能，而面向对象强调的是对功能的封装。面向对象是java的核心思想，这个思想来自于我们人的实际生活
  2. 面向对象的三大基本特征：封装(通过权限修饰符来实现)、继承(extends)、多态

成员变量和局部变量的区别

  1. 成员变量：直接放置在类中的变量称之为成员变量，与方法是平级关系
  2. 局部变量：在方法体中声明的变量称之为局部变量。
  3. 两者区别：
    1. 作用域区别：成员变量对于整个类有效、局部变量在某个范围内有效。
    2. 存储位置区别：成员变量随着对象的创建而存在，对象出现，成员变量出现，对象消失，成员变量消失，并且存储在堆内存中；局部变量在方法被调用，或者语句被执行的时候存在，存储在栈区内存中
    3. 初始值：成员变量有默认的初始值(引用类型是null，基本类型有各自默认的初始值)，局部变量必须初始化之后才能使用

静态变量和成员变量的区别

  1. 从调用方式来看：静态变量也称之为类变量，可以直接通过类名来调用，也可以通过对象名.来调用；成员变量只能通过对象名来调用
  2. 从存储位置来看：静态变量存储于方法区的静态区域；成员变量存储在堆内存中
  3. 从生命周期来看：静态变量随着类的加载而存在，随着类的消失而消失，生命周期很长；成员变量随着对象创建而创建，对象的消失而消失
  4. 与对象的相关性：静态变量是所有对象的共享的数据；成员变量是每个对象特有的

什么是匿名对象？匿名对象的应用场景

  1. 什么是匿名对象：没有名字的对象，是对象的一种简写形式

  2. 匿名对象的应用场景：

    1. 可以作为实际参数在方法中传递
    2. 只调用一次类的实例方法

字典树的实现

发表于 2019-08-02 | 分类于 算法总结

一些点

每个节点的内容

  1. 每个节点肯定得有一个存放字符的值

  2. 而且得有一个存放当前节点和其子节点的连接关系，要么是用map，要么是用数组

  3. 而且还有一个isWord用来表明从根节点到当前节点，是否是一个字符串

  4. 用map的好处就是能存储多种多样的字符串，只要能通过Character类型表达的，都能放入map中

往字典树中增加字符串

  1. 因为字典树的特性，往树中增加字符串必然要从根节点开始往下逐个插入字符
  2. 对于每个节点插入的操作：先看当前的节点的子树中是否存在要插入的字符，如果存在指针就指向节点的子节点，并且获得下一个要插入的字符，在进行同样的判断；如果说当前节点中不存在要插入的字符，就新造一个子节点，并且把该子节点与当前节点相连，同样的再更新指针的位置和要插入的字符，直到要插入的字符串都插入到了字典树中。
  3. 在字符串插入完毕之后，指针指向的应该是字符串的最后一个位置，这时可以把该节点的isWord设置为true表示从根节点到当前节点这一个字符串表示一个字符串。

查询字典树中是否存在某个字符串

  1. 同样因为字典树的特性，查询字典树中是否存在某个字符串也是得从根节点开始
  2. 对于每个节点的查询操作：看当前节点的子节点中是否存在要查询的字符，如果不存在就直接表明该字典树中不存在该字符串，如果存在当前字符的话，就继续从子节点的子节点继续查询是否存在字符串的下一个字符
  3. 在字符串的全部字符都查询过后，如果发现都存在，还得看指针当前指向的节点中的isWord是否为true。如果是true就表明字典树中存在该字符串，如果不存在就说明当前查询的字符串是字典树中某个字符串的前缀

简单的通配符 . (简单的正则表达式)

  1. 如果说.代表任意一个字符都能匹配的话，那么当某个节点遇到 “.” 这个字符的时候，就需要遍历当前节点的所有子节点，这里用递归就很方便了

String、StringBuffer、StringBuilder的区别

  1. String：当String字符串遇到频繁的修改的时候，每修改一次并且修改完的内容与之前不同时，虚拟机都会在堆区开辟新的内存，让栈区的内存去指向新开辟的堆区内存，这样会使得内存存在浪费。

  2. StringBuffer和StringBuilder字符串在遇到频繁的修改的时候，它只会在其本来所处的堆区内存中不断修改内容而不会频繁的开辟新内存，节约了空间。

  3. StringBuffer和StringBuilder的区别就是StringBuilder在多线程中会遇到线程安全问题，但是其在单线程中StringBuilder的效率是比StringBuffer要高的。

  4. StringBUffer和StringBuilder的使用取舍是当多线程的时候使用StringBuffer,单线程的时候使用StringBuilder。

ArrayList和LinkedList的区别

  1. ArrayList底层是数组(Object类型的数组)，LinkedList的底层是链表
  2. 由于ArrayList的底层是数组，因此查询数据比较快，但是对数据的增删比较慢，LinkList查询很慢，但是增删很快。

HashMap和Hashtable的区别

  1. HashMap和Hashtable的底层都是哈希表
  2. HashMap是jdk1.5之后出现的版本，Hashtable是在jdk1.5之前出现的
  3. Hashtable是线程安全的，HashMap是非线程安全的。
  4. HashMap的key和value可以为null，但是Hashtable的key和value不能为null

什么是编译时期的异常？什么是运行时期的异常

  1. 首先确立一个观点：写完的代码称之源文件(.java)，经过一系列工具(如idea，eclipse)的编译后成为java字节码文件(.class)，最后有Java虚拟机来执行字节码文件。
  2. 源文件编译成字节码文件时出现的异常是称之为编译异常(CE)，当程序被执行时发生的异常叫做运行时异常(RE)。
  3. 通常出现的编译时期异常有：FileNotFoundException(文件没有找到)、IOException(IO流异常)、ClassNotFoundException(没有找到类的异常)、EOFException(当用IO流读取文件的时候，文件的数据不存在会发生)、IllegalArgumentException(错误的参数，比如函数调用时传参)。
  4. 通常的运行时期异常有：NullPointerException(空指针)、ArithMeticException(比如10除以0的时候)、ArrayIndexOutOfBoundsException(数组越界)、StringIndexOutOfBoundsException(字符串下标越界)、IllegalArgumentException(RE时也有可能出现参数传递错误).

  1. 由字母、数字、下划线和$组成，不能以数字开头

java中的数据类型

  1. 数据类型包括基本数据类型和引用数据类型
  2. 基本数据类型包括：byte、boolean、char、short、int、long、float、double
  3. 引用数据类型包括：普通类、接口、数组

java语言的三种技术架构

  1. JAVAME：小型版本，是为开发电子消费产品和嵌入式设备提供的解决方案。手机
  2. JAVASE：java标准版，集合、多线程、面向对象等等
  3. JAVAEE：企业版，jsp、servlet什么的

重载与重写的区别

  1. 重载：同一个类中多个方法名字相同，但是参数类型，参数个数不同
  2. 重写：子类中的方法与父类中名字相同

内存结构

  1. 栈内存：用于存储局部变量，当数据使用完，所占用的栈区内存会被释放
  2. 堆内存：数组和对象，通过new建立的示例都放在堆区的内存中
  3. 方法区：静态成员、构造函数、常量池、线程池都是放在方法区的
  4. 本地方法区：window系统占用
  5. 主要就是分成这4类内存结构
