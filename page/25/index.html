二叉堆实现的优先队列

点

  1. 知道堆怎么实现后，再用堆去实现一个优先队列就很容易了

    1. 优先队列的add操作其实就是堆的增添一个元素的操作
    2. 优先队列的remove操作其实就是堆的取出最大元素的操作
    3. 还有诸多操作等等

  2. 注意此时实现的优先队列是有缺陷的，是没有办法使用比较器排序的，只能在类中实现compare方法来定义优先级，这就造成了存储基本类型时不能自定义优先级的不方便之处，也就是说存储基本类型时，是按照默认顺序来的

  3. 这里就有必要提及java本身的PriporityQueue，支持了自定义比较器，也就是说优先队列不管存储自定义类型还是基本类型的，都可以通过自定义比较器来实现自定义优先级，从而更方便的使用优先队列。

  4. 这里提一下java内置的优先队列的可以用比较器的方法来实现排序的原理：本质上Comparator比较器是一个接口，这个接口里面有一个方法是compare方法(接口里面自然是空方法)，然后在优先队列的构造方法里面需要一个接口的对象(肯定是匿名内部类)，在创建优先队列的对象的时候，传递一个重写了compare方法的Comparator接口的匿名内部类。在优先队列内部使用compare方法时只需要通过之前传过来的接口的对象就可以使用compare方法了。

二叉最大堆的实现

发表于 2019-08-20 | 分类于 算法总结

有必要提及的点

  1. Heapify这个操作是将一个数组转化为一个堆，这里的Heapify转化为一个最大堆，最大堆最小堆的转化原理都是一样的，都是寻找到最后一个非叶子节点的节点所在的索引(因为是数组实现的堆，所以会有节点对应的索引这么一说，其次这个索引为最后一个叶子节点的父节点的索引)，然后从这个索引开始往回一个节点一个节点的开始siftDown操作。
  2. 对于用数组实现的堆中求父节点和子节点的索引
    1. 当索引为0开始时：
      1. 父节点：(i-1)/2
      2. 左孩子：i*2+1
      3. 右孩子：i*2+2
    2. 当索引为1开始时：
      1. 父节点：i/2
      2. 左孩子：i*2
      3. 右孩子：i*2+1
  3. 对于往堆中添加元素e的时候，操作是
    1. 先把要添加的元素e放到数组的末尾
    2. 因为这时加入新元素可能会破坏堆的结构，所以要对置于末尾的元素e进行siftUp操作
      1. 对于siftUp操作来说(这里以最大堆来实现)：假设元素x(x会变的)进行siftUp操作
      2. 把x与x的父节点进行大小比较，如果父节点要比x小的话，就交换这两个节点的位置(在数组中的实现就是交换两个索引上的值)
      3. 交换完后，此时x为之前的x的父节点，继续对新的x进行siftUp。
  4. 对于往堆中取出最大元素时，操作是
    1. 先把要删除的元素的值先保留上
    2. 再把最后一个元素上的值与第一个元素上的值进行交换
    3. 然后删除最后一个元素
    4. 再对第一个元素进行siftDown操作
      1. 对于siftDown操作：假设元素x进行siftDown操作
      2. 每次寻找x的左右孩子节点的最大值(用最大值的原因是防止更新以后的x节点比左右孩子小)对应的索引
      3. 如果说x比两个孩子节点的值都大就说明堆的结构是完整的，就退出siftDown操作
      4. 如果进行到这一步就说明堆的结构是不正确的，这时交换x与第2步找到的索引上的值
      5. 更新x的索引，新索引就是第二步找到的索引
  5. 堆还有很多种，比如d叉堆，索引堆，二项堆，斐波那契堆

final关键字

  1. final是一个修饰符，可以修饰类、变量、方法
    1. 当修饰类时：当类被final修饰时，此类不能被其他类继承，
    2. 当修饰变量时：变量就成为一个常量，常量的值不能被修改
    3. 当修饰方法时：此方法不能被子类重写

Arrays、Math、Random等常用工具类

  1. 工具类：工具类一般不new对象(构造方法私有)，并且其中的方法基本都是静态的

  2. Arrays：sort()和asList()方法

  3. Math类：abs()求绝对值pow()求次方

instanceOf关键字

  1. 使用方法：对象 instanceOf 类型 ,返回值true表示对象是这个类型的，false则表示不是

多线程相关面试题

  2. 什么是线程：进程内部的一条执行路径或者一个控制单元
  3. 实现线程的几种方式：
    1. 让普通类继承Thread类

  4. run()与start()的区别：

字节流与字符流的区别

  1. 常见字节流，可以看出字节流都是以InputStream/OutputStream结尾的
  2. 常见字符流，可以看出字符流是Reader/Writer结尾的
  3. 两者区别的话
    1. 字符流主要用于读取纯文本，纯纯的文本
    2. 字节流主要用于读取图片，视频什么的，也能用于读取纯文本，底层肯定用的也是字符流的原理

什么是序列化

  1. 对象本身是不能直接保存到磁盘中的，如果把对象序列化之后，对象就可以保存的磁盘中了
  2. 直接让对象实现Serializable接口，然后生成序列号就OK

反射

    2. 对象.getClass()

  2. xml的约束文件：dtd、xsd，约束文件用来规定xml中哪些能写和哪些不能写
  3. 解析xml文件的几种方式：

leetcode-1013. 将数组分成和相等的三个部分

发表于 2019-08-03 | 分类于 leetcode

任务

示例

示例

示例

理解

  1. 最开始的想法：既然是把数组分成三个部分，那就那两个索引来分割整个数组成为三个部分，然后分别求三个部分的和，如果三个部分的和相同的话，就表明数组有和相同的三个部分，否则没有

    其中i,j就是根据一个二重循环拿到所有的可能的数组的三个部分

  2. 后来想到一个数组的和一定是一个定值，那么把这个定值三等分，得出一个temp，如果数组存在和相等的三个部分，那么每个部分的和一定是temp，很简单的一个道理，一开始没有想到，然后依照题意每个部分都是连续的，然后在从头遍历数组的时候找temp就行了，找完整个数组看看其中和为temp的个数是不是为3个。按照这个思路找数组中和相等的k个部分也是可以的。

代码

知识点

  1. python中直接求和是可以用sum函数的

  2. 双重循环不仅可以拿到数组的任意序列，还是两个点，把数组分成任意3个部分。
